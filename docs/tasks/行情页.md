# 行情页：涨跌停四池 + 连板梯队

**优先级**：P0
**状态**：待开始
**依赖**：`limit_stock_snapshot` 已就绪 ✓

## 背景

`limit_stock_snapshot` 表已存储涨停(up)/跌停(down)/连板(consec)/炸板(up_open) 四个池的数据，含原因、板块标签、连板天数等字段。需要提供 API 接口并在前端展示。

## 验收标准

- [ ] API 返回正确的四池数据，支持 `pool_type` 筛选
- [ ] 前端行情页展示涨停池：股票名、现价、涨跌幅、封板时间、原因、板块标签
- [ ] 连板梯队：按 `limit_days` 分组展示（2板/3板/N板）
- [ ] 跌停池：展示原因字段
- [ ] 炸板池：展示
- [ ] 文字使用中文

## 子任务

### 1. API 层
- [ ] `GET /api/market/limit-pool?pool_type=up&date=today` → 返回对应池数据
- [ ] `GET /api/market/limit-pool/summary` → 返回四池数量汇总
- [ ] 文件：`backend/src/app/api/market.py`

### 2. 前端类型定义
- [ ] `LimitStock` 类型定义（frontend/src/types/）
- [ ] API 请求函数（frontend/src/api/market.ts）

### 3. 前端页面
- [ ] 行情页 Tab 切换：涨停 / 跌停 / 连板 / 炸板
- [ ] 涨停池表格：股票名、现价、涨跌幅、封板时间、原因、板块 tag
- [ ] 连板梯队：按天数分组的卡片视图
- [ ] 文件：`frontend/src/pages/Market/`

## 接口设计

```
GET /api/market/limit-pool
Query: pool_type: up|down|consec|up_open (可选，不传返回全部)
       date: YYYY-MM-DD (可选，默认今天)
Response: {
  data: [
    {
      secu_code, secu_name, last_px, change,
      limit_time, reason, reason_detail,
      plate_codes, plate_names,
      is_st, limit_days, pool_type
    }
  ],
  summary: { up: 39, down: 23, consec: 0, up_open: 0 }
}
```
